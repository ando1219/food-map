# 🍜 大学城美食地图 - 潘柏如

> 帮助大学城高校联盟的同学快速发现周边美食，解决「今天吃什么」的世纪难题。

**在线访问** → [ando1219.github.io/food-map](https://ando1219.github.io/food-map)

---

## 功能介绍

- **按学校筛选** — 覆盖 A/B/C/D 四所高校，选择你的学校只看周边餐厅
- **按校区定位** — 支持主校区、北校区等多校区细分，精准缩小范围
- **输入当前位置** — 选择宿舍、教学楼、图书馆等具体位置，自动按步行距离排序
- **预算 & 菜系筛选** — 按人均消费和菜系快速过滤
- **随机推荐** — 选择困难？一键随机推荐一家
- **餐厅详情** — 查看推荐菜品、同学评价、步行时间

## 产品思路
### 一、背景与问题

大学城高校联盟（A/B/C/D 四所高校）周边聚集了大量餐饮商户，但学生普遍面临以下痛点：

| 痛点 | 具体表现 |
|------|----------|
| 选择困难 | 不知道今天吃什么，在宿舍纠结半小时 |
| 信息分散 | 口碑靠群聊，新生完全摸不到门路 |
| 缺乏学生视角 | 大众点评不标注离哪个校区近、不体现学生预算 |
| 跨校信息割裂 | 四所高校学生无法共享周边美食信息 |

**核心用户**：大学城内在校学生，尤其是大一新生、预算有限的同学。

---

### 二、产品定位

> 一个专为大学城学生设计的本地化美食发现工具——不是大众点评的缩小版，而是从学生视角重新组织信息。

**差异化核心**：
- 以「学校 → 校区 → 当前位置」三层定位替代泛地图
- 以「学生预算标签」替代人均消费区间
- 以「同学说」替代匿名评价，增强信任感
- 以「随机推荐」直接解决选择困难

---

### 三、功能结构

```
大学城美食地图

│   └── 校区细分（主校区 / 北校区 / 东校区…）
├── 当前位置输入
```

### 四、核心设计决策

#### 1. 三层定位：学校 → 校区 → 具体位置

普通地图应用要求用户自己搜索位置，对本地新生不友好。本产品将定位拆成三层递进：

- **学校**：过滤掉其他学校周边无关餐厅
- **校区**：同一所学校可能有多个校区，进一步缩小范围
- **具体位置**：选择宿舍、教学楼等，按真实步行距离排序

这样即使是第一天入学的新生，也能快速找到离自己最近的饭馆。

#### 2. 「随机推荐」作为核心功能

针对「选择困难」这一最高频痛点，设计了随机推荐按钮。它不是完全随机，而是在当前筛选条件（学校 + 预算 + 菜系）的范围内随机，保证推荐结果是用户能接受的选项。

#### 3. 学生预算标签

将价格分为三档（¥10以下 / ¥10–20 / ¥20–30），符合学生实际消费习惯，比「人均消费」更直觉。同时在卡片上同时展示预算标签和「热门」标签，帮助用户快速判断。

#### 4. 「附近」标签强化距离感知

输入位置后，步行距离较近的餐厅会额外打上绿色「附近」标签，让用户在浏览卡片时一眼就能识别，不需要逐个点开查看距离。
